# 12주차 TIL: 앱 기본 구조 및 인증/루트 레이아웃 구현

**학습 주제:** 스플래시 화면, 로그인(인증) UI, Root 레이아웃(하단 탭) 구성

## 1. 주요 학습 내용 (강의 12주차 진도)

이번 주차에는 앱의 진입 시점부터 메인 화면으로 넘어가는 전체적인 흐름(Flow)을 구성했습니다. 강의 가이드에 따라 Firebase 연동은 제외하고, UI와 로직 위주로 **목업(Mock-up)** 처리를 진행했습니다.

### A. 프로젝트 환경 설정 (Review)
* **패키지 설치:** `pubspec.yaml`에 필수 라이브러리를 세팅했습니다.
    * 상태 관리 및 라우팅: `get`
    * 에셋 설정: 이미지를 사용하기 위해 assets 폴더 경로를 등록했습니다.

### B. 스플래시 페이지 (Splash Page)
* **개요:** 앱 실행 시 데이터를 로딩하는 동안 보여줄 시작 화면을 구현했습니다.
* **구현:** `GetXController`를 활용하여 `SplashController`를 만들고, 1.5초 후 자동으로 로그인 화면으로 이동하는 로직을 비동기(`Future.delayed`)로 처리했습니다.

### C. 인증 프로세스 (Login Page - 손코딩)
* **UI 구성:** 교재의 가이드에 따라 "당신 근처의 밤톨마켓" 문구와 소셜 로그인(구글, 애플) 버튼이 있는 로그인 화면을 손코딩으로 구현했습니다.
* **로직:** 실제 로그인은 구현하지 않고, 버튼 클릭 시 메인 화면(`Root`)으로 넘어가는 흐름을 `Get.offAll()`을 사용하여 구현했습니다. (뒤로가기 방지)

### D. 앱 Root 레이아웃 (Root Layout)
* **BottomNavigationBar:** 앱의 골격이 되는 하단 탭바를 구성했습니다.
* **상태 관리:** `BottomNavController`를 생성하고 `RxInt`로 현재 인덱스를 관리하여, 탭을 누를 때마다 화면(`IndexedStack` 또는 `Obx`)이 즉시 전환되도록 구현했습니다.
